# Contributing to ask

Thank you for your interest in improving ask :)  Since this is a small bash script we want to keep it as simple as possible.

# 1. Found a Bug or Have an Idea?

Opening an Issue on GitHub first to discuss it before you spend time writing code would be appreciated.

Include your Bash version, curl and jq version and the exact error message or unexpected behavior.

# 2. Making Changes (Pull Requests)
If you want to contribute code, please follow this workflow:

Fork the repository.

Clone your fork locally.

Branch: Create a new branch for your feature or bug fix (git checkout -b feature/my-cool-idea).

Commit: Write clear, concise commit messages explaining why you made the change.

Push: Push your branch to your fork.

Pull Request: Open a PR against the main repository.

# 3. Coding Guidelines

Dependencies: Do not add any new dependencies.

Style: Write clean, readable Bash. Comment any complex jq parsing logic so others can understand it.

Testing: Before submitting a PR, test your changes using standard command-line arguments and piped input.
